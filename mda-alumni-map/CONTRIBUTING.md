# Руководство по разработке (Contributing)

Мы рады любым улучшениям проекта! Пожалуйста, следуйте этим правилам:

## Как внести вклад

1.  Сделайте Fork репозитория.
2.  Создайте ветку для вашей фичи (`git checkout -b feature/AmazingFeature`).
3.  Зафиксируйте изменения (`git commit -m 'Add some AmazingFeature'`).
4.  Отправьте ветку в ваш репозиторий (`git push origin feature/AmazingFeature`).
5.  Откройте Pull Request.

## Стандарты кода

- **TypeScript:** Обязательно использование строгой типизации. Проверка: `npx tsc --noEmit`.
- **Компоненты:** Используйте Composition API и `<script setup>`.
- **Стили:** Только Tailwind CSS. Не пишите кастомный CSS без крайней необходимости.
- **Тесты:** Если вы добавляете новую логику в API или валидаторы, добавьте соответствующий тест в `server/tests` или `src/__tests__`.

## Сообщения о багах

Используйте GitHub Issues для сообщений о проблемах. Обязательно прикладывайте шаги для воспроизведения и описание ожидаемого поведения.
